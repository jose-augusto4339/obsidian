## A fase de  projeto de software

Durante essa etapa, é criada uma arquitetura de software que serve como base para o desenvolvimento. O objetivo é garantir que o software final tenha eficiência, flexibilidade e atenda a necessidade dos usuários.
Diversas atividades são feitas para garantir que o software atenda as expectativas:
- Definir a arquitetura do sistema
- Projeto dos componentes e módulos do sistema
	- Os componentes são projetados para serem coesos e com baixo acoplamento, facilitando a manutenção e evolução do software
-  Especificação de como os diferentes componentes se comunicarão 
- Modelagem de como os dados serão armazenados, acessados e manipulados
- Criação de esboços iniciais e protótipos 

A fase de projeto termina na produção de uma documentação detalhada que serve como base sólida para a implementação.

## Conceitos básicos de projeto de software

## Modelo do Projeto

É uma representação do sistema que será desenvolvido. Um mapa que orienta a implementação do software.
Especifica a estrutura dos diferentes componentes e como interagem entre si.
Inclui várias metodologias técnicas fundamentais, como UML com diagramas de classe ou sequencia para representar de forma clara e concisa a estrutura e o comportamento do software. Modelos arquiteturais definem a estrutura geral do sistema, descrevem a organização dos componentes e suas interações.

Exemplo de modelos arquiteturais:
- Camadas:
	- O sistema é dividido em diferentes níveis de funcionalidade.
- Micro-services:
	- Criação de pequenos serviços independentes que se comunicam entre sí.

Os protótipos são representações simplificadas do software. São úteis para capturar feedback dos usuários e ajustar o projeto.

Os Modelos de Fluxo de Dados são usados para descrever como os dados circulam pelo sistema. Crucuais para garantir que todos os aspectos do processamento de informações ejam considerados e corretamente implementados.
