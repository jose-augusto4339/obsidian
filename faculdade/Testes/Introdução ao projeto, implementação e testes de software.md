## Projeto de software
Fase inicial e uma das mais críticas. Estágio onde é definida a arquitetura. Um bom projeto pode reduzir significativamente os custos de desenvolvimento, facilitar a manutenção e garantir que o software seja escalável e flexível para futuras atualizações.
Nesse processo são feitas decisões como escolha de padrões de design, seleção de tecnologias, frameworks e ferramentas, e a definição de metodologias de desenvolvimento.

Objetivos do projeto de software:
- Arquitetura
	- Desenvolvida de maneira a dividir o sistema em módulos ou componentes independentes.
- Linguagem
	- A seleção deve ser feita cuidadosamente, considerando fatores como desempenho, compatibilidade e curva de aprendizado.
- Integração
	- As integrações entre diversos componentes deve ser planejada para minimizar riscos de incompatibilidades e garantir funcionamento harmonico.

## Implementação de software
Processo no qual as ideias e especificações elaboradas na fase de projeto são traduzadas em código executável. Etapa onde o conceito vira produto, e a eficácia do projeto é verificada.
A implementação exige conhecimento técnico, disciplina e aderência a boas práticas de programação.
A escrita do código é o ato de traduzir as especificações de design em código-fonte, utilizando as linguagens de programação e frameworks escolhidos na fase de projeto.

Princípios e práticas que devem ser seguidos durante a escrita do código:
- Modularidade: 
	- Dividir o código em pequenos módulos ou funções, cada um com uma única responsabilidade.
- Clareza e simplicidade:
	- Deve ser claro e simples de ler e enteder. Utilizar nomes descritivos para variáveis, funções e classes.
- Consistência:
	- Seguir convenções de código consistentes ao longo de todo o projeto.
- Boas práticas de programação: 
	- Práticas como o uso de padrões de design, tratamento adequado de erros e otimização de desempenho.
- Testes automatizados:
	- Desenvolver testes automatizados, como testes unitários.

### Documentação
Manter uma documentação clara e detalhada do código permite que outros desenvolvedores compreendam facilmente como o software foi construído e como deve ser mantido.

Os principais tipos de documentação incluem:
- Documentação de código: Inclui comentários no código-fonte
- Documentação técnica: Detalha a arquitetura geral do sistema, descreve os principais componentes, as tecnologias utilizadas e as dependências externas. Também inclui instruções sobre como configurar o ambiente de desenvolvimento, como construir e implantar o software e como realizar testes.
- Documentação do usuário: Importante para garantir que aqueles que usarão o software saibam como opera-lo.

## Integração Contínua (CI)
Fundamental para desenvolvimento de software ágil e eficiente. Automatiza as etapas de construção, teste e implantação, permite que as equipes integrem suas mudanças de código com frequência. Agiliza o processo de desenvolvimento, garante a qualidade do software e proporciona um feedback constante aos desenvolvedores.

## Teste de software
Uma das fases mais críticas e da talhadas do ciclo de vida do desenvolvimento. Deve garantir que o software funcione conforme o esperado e que não contenha falhas que comprometam a experiência do usuário ou a segurança dos dados.
Os testes devem ser realizados de maneira sistemática e abrangente, cobrindo todas as partes do sistema e todos os cenários de uso possíveis.
Os testes não apenas identificam erros e inconsistências, mas também validam se o software atende aos requisitos definidos na fase de projeto.

Tipos de teste:

- Teste unitário: 
	- Verifica a funcionalidade de componentes do sistema. Fundamentais para identificar erros logo no início do processo de desenvolvimento.
- Teste de integração:
	-  Avalia a integração entre os diferentes módulos do sistema. Visão problemas de integração que podem não ser aparentes nos testes unitáios
- Teste de sistema:
	- Valida se o software atende aos requisitos funcionais e não funcionais especificados.
- Teste de aceitação:
	- Realizado pelo cliente ou usuário final. Verifica se o software atende às necessidades e expectativas de que irá utiliza-lo.
